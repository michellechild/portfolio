Rethinking the payment portal

A streamlined payment process for a seamless user experience.

ccepTx Pro is a software that helps orthodontists present treatment and fees to their patients.

When the development team was ready to integrate a payment portal into the application, I saw this a great opportunity to look at the treatment and payment plan process as a whole and redesign the experience from a user perspective.

I've started with a summary of the project. Read on to see the details, including before and after screens.

Download the InVision prototype (16 MB).

The Issues

  • Lacking in focus

    Where should I be looking? What should I do first? The existing experience felt scattered, with a lot of important information given to the user at once.

  • No end in sight

    This feels long - How long will this take me to complete? Users often stopped midway and finished the process over the phone.

  • Outdated design

    Is this website credible? Users weren't convinced.

  • Confusing Instructions

    What does this mean? Offices often had calls from customers who didn't finish because they weren't confident they understood the terms.

  • The Solution

    Define the action

    Clarify the purpose of each page. Intuitively guide the user to complete each defined action.

  • Display the start and end points

    Encourage the user by showing finish line. Showing the whole picture makes each step easier to understand.

  • Build trust

    Reflect brand values with consistent, modern design. Incorporate proven UX principles to gain the user's trust.

  • Provide further explanation

    Rewrite instructions to be clear & concise. Utilize tooltips and footnotes to further explain the details.

The Plan

The business analyst and I reviewed the existing screens and grouped them by action, allowing us to see distinct steps of the process.

The Execution

The structure was in place and ready for some design love. I took some time to research best practices on UI Patterns, UX Booth and a few others - then went to work.

Steps 1-2 of 5: Before

  • Understated primary action

    The required action on this page feels more like an alert than the primary focus.

  • Does the sidebar add value?

    Although this is a customer-facing page, the sidebar text was directed toward the Treatment Coordinator, an office employee.

  • Why show PDFs?

    These forms, while pertinent, take up a majority of the screen. The user must go to a different screen to eSign the forms - is there a better way to display them here?

Steps 1-2 of 5: After

  • 'Accept Treatment Presentation' step

    Accepting the treatment has been an isolated step - until now. The user can now view the treatment plan in one place, and easily revisit the presentation at any time.

  • Restructured form section

    By removing the iframe, the screen maintains focus. No functionality is lost - user is still able to view, download and print from this screen.

  • External link icon & tooltip

    To complete the step, the user will be automatically directed to an eSign portal. The icon and tooltip help explain this detail.

Step 3 of 5: Before

  • Inconsistent styling

    The 'next step' link has changed from a green button to a link, making it more difficult for the user to find what to do next.

  • Why use a modal?

    Again a problem with inconsistency - why does this date picker need to be in a modal? Secondly, the Appointment Date input field looks disabled initially - it is not.

Step 3 of 5: After

  • Consistent Layout

    The page layout and elements are consistent throughout the steps. This allows the eye to quickly focus on new or changed text or elements.

  • Consistent date input

    Corrected inconsistencies with date formatting and coloring to eliminate confusion.
